# Tool-version catalog consumed by `tools.just` recipes
# (_ox-check-require, ox-check-setup, ox-check-tools-check).
#
# Format: <cargo-subcommand-binary>=<pinned-version>
# One per line. Lines starting with `#` and blank lines are ignored.
# Versions are SemVer; comparison uses `[version]` (so "1.2.10" > "1.2.9").
#
# Version semantics:
#   - On install (`ox-check-setup`): exactly this version (`=<pin>`).
#     Pinning the install avoids the CI reproducibility risk of
#     "upstream release between yesterday's green build and today's
#     PR breaks things" (cargo-spellcheck 0.15.7's em-dash regression
#     is the canonical example of why >= is the wrong policy here).
#   - On runtime check (`_ox-check-require`): the installed version
#     must be `>= <pin>`. A user who has manually upgraded a tool
#     for their own reasons is not downgraded by setup.
#
# To bump a pin, edit the version in place and re-run `cargo ox-check
# update`. The dirty-file flow preserves the edit on subsequent runs.
# To add a new tool, append a line. To remove a tool, delete its line
# (and any recipe in checks.just that depends on it).
#
# Non-catalog dependencies (rustc, pwsh, just itself) are handled
# directly in tools.just rather than through this file -- their
# install/upgrade paths are not cargo-managed.
